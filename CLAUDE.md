# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Overview

A dotfiles repository managed by GNU Stow.

## Usage

Managed via Makefile (`make` shows usage). Packages are auto-discovered
(every top-level directory).

- `make install` — stow all packages into ~
- `make update` — restow after adding/removing files
- `make preview` — dry-run
- `make uninstall` — remove all symlinks

## Commit messages

The commit messages adhere to the following format:

Capitalized, short (50 chars or less) summary.

More detailed explanatory text, if necessary. Wrap at 72 characters.
The blank line separating the summary from the body is critical —
tools like rebase can get confused if you run the two together.

Write in the imperative: "Fix bug" not "Fixed bug" or "Fixes bug."
This matches commit messages generated by git merge and git revert.

Describe what changed at the right level of abstraction — not so
broad it says nothing, not so specific it names every symbol touched.
"Add unary operator tokens" is better than "Add TokenMinus and
TokenTilde to the lexer".

Further paragraphs come after blank lines.

- Bullet points are okay, too
- Use a hyphen, followed by a single space, with blank lines in between
- Use a hanging indent
